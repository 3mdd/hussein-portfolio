AIRS — Automated Inspection & Reporting System

Overview

AIRS helps streamline inspection reporting by allowing inspectors to upload inspection info and photos, then generate standardized reports. Reports go through an approval process to maintain quality and consistency.

Status: 95% Completed Project

GitHub: https://github.com/3mdd/AIRS

Roles & Responsibilities

  • Inspector: upload inspection info/photos, add remarks/observations, generate report.
  • Manager: approve or reject reports.
  • Admin: manage users and admin tasks.

Key Features

  • Asset register and inspection history
  • Upload and manage inspection photos
  • Standardized report generation
  • Approval workflow (approve/reject)

ZZ Restaurant Ordering System

Overview

A restaurant ordering and delivery system with separate interfaces for customers, staff, and runners. Customers place orders, staff confirm/cancel and assign platforms, and runners update delivery status.

Customer / Staff / Runner Web-based

GitHub: https://github.com/3mdd/DAD-Project

Tech Stack

Frontend: HTML, CSS, Vanilla JavaScript
Backend: PHP
Database: MySQL

Key Features

  • Menu browsing + cart + checkout
  • Order tracking + order history
  • Staff dashboard to manage orders and menu items
  • Runner delivery updates (picked up → delivered)

UTeM Rooms/Halls Booking System

Overview

A booking system for UTeM facilities (rooms/halls) to reduce double booking and improve scheduling. Users can view availability, book by time slot, and manage bookings. Admin manages rooms and generates reports.

GitHub: https://github.com/3mdd/utem-room-booking-system

Tech Stack

Language: C++
Database: MySQL
Docs: Final report + slides (in repo)

Key Features

  • Register / login
  • View room details and availability
  • Book, update, or cancel bookings
  • Admin: manage rooms, view users, generate weekly/monthly reports
